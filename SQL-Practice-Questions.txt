TIL:
1. In SQL, indentation does NOT matter at all so, we'll use Python Syntax Structure so as to learn 2-for-1.
2. The semicolon tells the SQL engine where one command ends and the next one starts.

Guiding principles Python-style

4 spaces per indentation level.
One “block” per keyword (SELECT, FROM, WHERE, JOIN, etc.) on its own line.
Nested blocks (like subqueries) get indented one level deeper.
Closing parenthesis lines up with the start of the statement that opened it
Commas at line ends 
Semicolon ends the statement

9. Show the first_name, last_name, and height of the patient with the greatest height.

Approach 1. Best for handling ties(will display patients with the same height)

SELECT
    first_name,
    last_name,
    height
FROM
    patients
WHERE
    height = (
        SELECT
            MAX(height)
        FROM
            patients
    );

Approach 2. Best for speed

SELECT
    first_name,
    last_name,
    height
FROM
    patients
ORDER BY
    height DESC
LIMIT
    1;
